Les données du type tuples

Tuples vs Listes

Les tuples (ou uplets ou, p-uplets ou n-uplets) sont des séquences, tout comme les listes. Les tuples utilisent des parenthèses(*), alors que les listes utilisent des crochets.

Mais pourquoi des Tuples quand nous avons des listes  ?

La réponse est simple : les tuples sont plus rapides que les listes. Si on définit  un ensemble constant de valeurs qu’on souhaite simplement itérer, on utilise  Tuple au lieu d'une liste. 

Les éléments d'un tuple peuvent êtres hétérogènes. Cela veut dire qu'un tuple peut avoir n'importe quel nombre d'éléments et ils peuvent être de différents types (entier, flottant, liste, chaîne , etc.).

Un tuple ne peut pas être changés contrairement à une liste. On parle d’une séquence immuable. Cela signifie que les éléments d'un tuple ne peuvent pas être modifiés ou supprimés une fois qu'il a été crée. Par contre, si l'élément est lui-même un type de données muables comme une liste, ses éléments imbriqués peuvent être modifiés.


Les éléments d'un tuple sont immuables mais, la suppression d'un tuple est entièrement possible en utilisant le mot clé del. (voir l'exemple 8)
